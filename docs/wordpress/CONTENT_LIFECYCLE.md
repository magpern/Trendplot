# Content Lifecycle

How Trendplot should manage content over **years** on a connected WordPress site. The plugin supplies authoritative inventory, search, and metadata; Trendplot applies policy on top.

**Related:** [CONNECTOR_INVENTORY_SCHEMA.md](./CONNECTOR_INVENTORY_SCHEMA.md), [DRAFT_PUBLISHING_CONTRACT.md](./DRAFT_PUBLISHING_CONTRACT.md), [CONNECTOR_API_CONTRACT.md](./CONNECTOR_API_CONTRACT.md) (`GET /content/search`).

---

## Design principle

Trendplot **pulls** lifecycle signals (inventory + search). The plugin **does not** drive editorial workflows. No event bus required for lifecycle decisions in V1.

---

## Lifecycle categories

| Category | Definition | Trendplot action |
|----------|------------|------------------|
| **New content** | No matching post/product topic in search | Ideate → schedule → generate → `POST /drafts` |
| **Existing content** | Match found via `/content/search` or inventory | Do not duplicate; link or refresh instead |
| **Refresh candidates** | Published content stale by age or quality | Recommend update job; `PATCH /drafts` or operator review |
| **Expansion candidates** | Topic covered thinly (low `word_count`) or missing subtopics | Recommend new section or sibling article |
| **Follow-up articles** | Cluster has anchor piece but missing comparison/how-to/storage variant | Ideation with `recommendation_type=expand` |
| **Product-linked content** | Article tied to SKU via relationships | Maintain via `POST /relationships`; ideation uses `related_products` |
| **Duplicate detection** | Same `content_hash` or high title/keyword overlap | Block or merge recommendations |
| **Retirement / archival** | Post `trash`, `draft` abandoned, or product discontinued | Stop refresh; exclude from ideation; optional redirect note in metadata |

---

## Questions the plugin helps answer

| Question | Data source |
|----------|-------------|
| **Do we already have this article?** | `GET /content/search?query=...` + inventory title/slug match |
| **Was it generated by Trendplot?** | `trendplot_generated` / `generated_by_trendplot` on inventory & search results |
| **Is it stale?** | `modified_at`, `generated_at`, `last_ai_refresh` vs workspace policy (e.g. >12 months) |
| **Should it be updated?** | Refresh rules + `content_hash` change after re-generation |
| **Should it be expanded?** | Low `word_count`, missing H2 topics vs ideation abstract, expansion opportunity score |
| **Is there a related product?** | `related_products`, `primary_product`, product inventory + relationships |

## Recommended metadata (WordPress post meta)

Stored on Trendplot-managed posts (and exposed in inventory/search):

| Meta key | Purpose |
|----------|---------|
| `recommendation_id` | Lineage to recommendation / content-plan item |
| `opportunity_id` | AI ideation opportunity id when applicable |
| `generated_at` | ISO8601 first or last Trendplot generation |
| `content_hash` | SHA-256 of normalized body — duplicate detection |
| `trendplot_generated` | boolean — `true` for Trendplot-originated posts |
| `related_products` | Woo product IDs or slugs |
| `last_ai_refresh` | ISO8601 last Trendplot refresh/regeneration (optional) |

See [DRAFT_PUBLISHING_CONTRACT.md](./DRAFT_PUBLISHING_CONTRACT.md) for write requirements on new drafts.

## Example lifecycle flows

### Flow A — New article (no duplicate)

1. Ideation proposes “BPC-157 Storage Guide”.
2. `GET /content/search?query=BPC-157+storage` → empty or weak match.
3. Schedule → generate → `POST /drafts` with full metadata.
4. `POST /relationships` → BPC-157 + Bacteriostatic Water.
5. Inventory sync picks up new post with `generated_by_trendplot=true`.

### Flow B — Duplicate blocked

1. Ideation proposes “Bacteriostatic Water Storage Tips”.
2. Search returns existing post with high `match_score` and same `related_products`.
3. Trendplot marks recommendation `duplicate_blocked`; suggests refresh (Flow C) instead.

### Flow C — Refresh existing

1. Inventory flags post: `trendplot_generated=true`, `modified_at` > 12 months.
2. Reassessment creates **refresh candidate**.
3. Regenerate → `PATCH /drafts/{id}`; update `content_hash`, `last_ai_refresh`.
4. No new URL; relationships unchanged unless products changed.

### Flow D — Follow-up in product cluster

1. Search shows Reconstitution + Storage guides exist for BPC-157.
2. Gap: no “BPC-157 vs TB-500” article.
3. Ideation creates follow-up recommendation only for missing slot.
4. New draft + relationship merge on product.

### Flow E — Expansion (thin content)

1. Search finds “BPC-157 Overview” with `word_count` < 800.
2. Expansion candidate → regenerate with section expansion or PATCH.
3. `last_ai_refresh` updated; calendar notes expansion not net-new URL.

### Flow F — Retirement

1. Product discontinued in Woo → inventory marks product `status` inactive.
2. Trendplot stops ideating new BPC-157 articles.
3. Existing posts: optional `noindex` / update disclaimer via operator; exclude from refresh queue.

## Workflows over time

### Year 1 — Net new coverage

1. Full `GET /inventory` on connect
2. Ideation avoids duplicates via `/content/search`
3. New drafts carry full metadata
4. `POST /relationships` links products

### Year 2+ — Refresh and expansion

1. Scheduled inventory sync (`updated_after` cursor)
2. Trendplot reassessment run flags **refresh candidates** (`modified_at` + traffic if available later)
3. Operator or auto policy: create refresh recommendation → regenerate → `PATCH` existing post id
4. Update `content_hash`, `generated_at`, `last_ai_refresh`

### Clustering (BPC-157 example)

**Product:** BPC-157  

**Related articles (target cluster):**

- BPC-157 Reconstitution Guide
- BPC-157 Storage Guide
- BPC-157 vs TB-500

Trendplot uses `/content/search?query=BPC-157` + `related_products` to see which cluster slots are filled vs gaps for new ideation.

## Duplicate detection policy

| Signal | Threshold (configurable) |
|--------|---------------------------|
| Exact `content_hash` match | Block generation |
| Title similarity + same `related_products` | Warn / merge recommendation |
| Same `recommendation_id` on new job | Idempotent — update existing post |

---

## Refresh vs new draft

| Condition | Action |
|-----------|--------|
| Same post, new generation | `PATCH /drafts/{id}` + update meta |
| Different post, same topic | Prefer refresh of canonical URL (highest traffic or oldest Trendplot post) |
| No post id resolvable | `POST /drafts` new + redirect/canonical policy (operator) |

## Without plugin (fallback)

Crawl inventory lacks reliable `trendplot_generated` and `content_hash`. Lifecycle quality is **degraded** until connector Phase 1 is live.
